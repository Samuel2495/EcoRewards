# EcoRewards Backend Requirements

## Functional Requirements
- User Authentication (Signup/Login/Profile)
- Role-based Access Control (User/Admin/Hub-Admin)
- Disposal Hub Management (CRUD + Geolocation search)
- Eco-Task Submission & Verification Workflow
- Bulk Pickup Request Management
- Points & Credits Calculation Logic
- Hub Check-in (QR Code verification)
- Leaderboard Ranking

## Suggested Node.js Tech Stack (Express/Next.js)
The following dependencies are highly recommended:

```text
express             # Web framework
mongoose            # MongoDB ODM (or 'pg' for PostgreSQL)
bcryptjs            # Password hashing
jsonwebtoken        # User authentication (JWT)
cors                # Cross-Origin Resource Sharing
dotenv              # Environment variable management
multer              # For image uploads (if adding task photos)
express-validator   # Data validation
```

## Security Requirements
- All passwords must be hashed before storage.
- Private routes must be protected by JWT middleware.
- Admin routes must check for the `role` field.
- Hub-specific data must be scoped to the `hubId`.
